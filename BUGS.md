Issue 1 — Double-path bug (/home/kamau/.../home/kamau/.../outputs/...)
Root cause: OUTPUTS_DIR returns an absolute path, and somewhere in promo.js or compositor.js a second path.join(OUTPUTS_DIR, absolutePath) call concatenates it again.
Fix: In src/engines/promo.js brandImage(), change return { file: outFile, ... } — the file field is already an absolute path, nothing to fix there. The real double-path is in cache_stats and fetch_optimal_media — the assets/external_cache dir is being joined onto OUTPUTS_DIR instead of the engine root. Fix: in src/engines/media-cache.js, replace path.join(cfg.OUTPUTS_DIR, 'assets/external_cache') with path.join(__dirname, '..', '..', 'assets', 'external_cache').

Issue 3 — ask_claude and generate_video_script fail
Root cause: Both call the Anthropic API internally. The aiConfigured: true in get_api_status only checks that the key exists in .env, not that it's valid. The key is expired/invalid.
Fix: Run tmux send-keys -t lavira to open the .env, replace ANTHROPIC_API_KEY with a fresh key from console.anthropic.com.
Issue 4 — get_destination_rotation_status returns raw MCP envelope
Root cause: That handler returns { content:[{type:'text', text: JSON.stringify(...)}] } — it wraps its own response in MCP format, which then gets double-wrapped by the server's main serializer.
Fix: In server.js case get_destination_rotation_status, change the return to just return the parsed array directly instead of the content envelope.
Issue 5 — smart_generate misidentifies destination ("Ol Pejeta" → "Nakuru")
Root cause: intent.js NLP doesn't have "Ol Pejeta" in its keyword map.
Fix: Add 'ol pejeta': 'Ol Pejeta', 'pejeta': 'Ol Pejeta' to the destination keyword map in src/orchestrator/intent.js.
